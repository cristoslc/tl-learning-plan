# Tech Lead Learning Plan

A capability-based development program for Tech Leads transitioning from application-level expertise to architectural thinking. Designed to be adapted by any engineering manager for their team.

**[View the interactive capability map](https://cristoslc.github.io/tl-learning-plan/)**

## What's in here

All curriculum content lives in [`dist/`](dist/).

- **Capability map** (`dist/_0 concept-map.md`) -- 11 capabilities that change how TLs show up in architectural conversations, plus an AI-as-thinking-partner thread. **Start here**, or use the [interactive HTML version](https://cristoslc.github.io/tl-learning-plan/).
- **Capability deep-dives** (`dist/capability-*.md`) -- one page per capability with situation, what changes, readiness check, curated resources, and a practice exercise
- **Media summaries** (`dist/summaries/`) -- structured notes from conference talks, podcasts, and interviews referenced across the program

## How to use this

1. **Read the capability map** to see the full landscape.
2. **Pick a capability** that matches where your team is right now, or start with Capability 1 and work through in order.
3. **Use discussion, not tests.** Each capability has discussion questions designed for 1-on-1s or small-group sessions. A capability is "done" when the TL can hold a substantive conversation about it, not when they've read the material.

## Structure

```
dist/                    # The curriculum (what TLs read)
  _0 concept-map.md      # Capability map -- start here
  capability-*.md        # Individual capability deep-dives (11 files)
  capability-map.html    # Interactive version with filters and progress tracking
  summaries/             # Media summaries
```

## Design principles

- **Capability-based, not topic-based.** Each capability describes a situation, what changes after learning it, and a self-check -- not a list of topics to cover.
- **Discussion-driven assessment.** Shallow answers to discussion questions are the signal to dig deeper. There are no quizzes.
- **Curated, not comprehensive.** Each capability has a "Start Here" (one or two resources) and a "Go Deeper" section. The goal is to get the idea across fast, not to assign a reading list.
- **AI-assisted learning.** The program treats AI tools as vocabulary accelerators: use them for first-pass exploration, then read the source material for the actual argument with its evidence and caveats.
